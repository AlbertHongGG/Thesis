# 任務 Spec：個人記帳＋月報表網站
Personal Expense Tracking & Monthly Report System

---

## 1. 產品目標（Product Goal）

建立一個供「個人使用」的線上記帳系統，解決以下核心問題：

- **日常財務追蹤**：讓使用者能快速、方便地記錄每日收入與支出
- **財務可視化**：透過日期分組、類別統計與視覺化圖表呈現財務狀況
- **報表分析**：自動產生月報表，協助使用者掌握每月財務變化

### 系統必備能力

- 使用者認證與資料隔離（每位使用者僅能存取自己的資料）
- 帳務 CRUD（新增、讀取、編輯、刪除）
- 類別管理（預設類別 + 自訂類別 + 停用機制）
- 日期分組與每日統計
- 月報表聚合與圖表（圓餅圖、長條圖）
- 資料一致性（報表、圖表、列表數據需一致）
- 匯出功能（CSV，選擇性）

## 2. 使用者角色定義（Roles）

### 2.1 訪客（Guest）

- **權限**：僅能存取公開頁面（登入頁、註冊頁）
- **可執行行為**：
  - 瀏覽登入頁
  - 瀏覽註冊頁
  - 執行註冊流程
  - 執行登入流程
- **限制**：
  - 不可存取帳務列表頁
  - 不可存取月報表頁
  - 不可存取類別管理頁
  - 嘗試存取受保護頁面時，導向登入頁

### 2.2 使用者（User）

- **權限**：完整存取自己的資料與所有功能頁面
- **可執行行為**：
  - 新增、編輯、刪除帳務
  - 檢視帳務列表（依日期分組）
  - 管理類別（新增、編輯、停用）
  - 查看月報表與統計圖表
  - 切換年份/月份查看歷史報表
  - 匯出當月帳務資料（CSV）
  - 登出
- **限制**：
  - 僅能操作自己的資料
  - 無法存取其他使用者的帳務或類別

> ⚠️ 本系統為「單人系統」，**無管理員角色**。每位使用者的資料完全隔離。

## 3. 使用者流程（User Flow）

### 3.1 訪客（Guest）流程

1. 開啟網站，進入登入頁
2. 選擇「註冊」進入註冊頁
3. 填寫 Email 與密碼，送出註冊
4. 註冊成功後自動登入，導向帳務列表頁
5. （或）直接在登入頁輸入帳密登入

### 3.2 使用者（User）流程

#### 主要流程：記帳

1. 使用者登入後進入帳務列表頁
2. 點擊「新增帳務」按鈕
3. 填寫帳務表單（類型、金額、類別、日期、備註）
4. 送出表單，新帳務出現在列表中
5. 可點擊編輯既有帳務
6. 可點擊刪除帳務（需二次確認）

#### 次要流程：查看報表

1. 從導覽列進入月報表頁
2. 預設顯示當月報表
3. 可切換年份/月份查看歷史資料
4. 查看總收入、總支出、淨收支
5. 查看類別圓餅圖（支出分布）
6. 查看每日收支長條圖

#### 次要流程：類別管理

1. 從導覽列進入類別管理頁
2. 查看現有類別清單（包含預設與自訂）
3. 可新增自訂類別
4. 可編輯類別名稱
5. 可停用類別（停用後不可用於新帳務，但歷史資料保留）

#### 次要流程：匯出資料

1. 在月報表頁點擊「匯出 CSV」
2. 下載當月帳務資料

## 4. 功能需求（Functional Requirements）

### 4.1 帳號與認證（Authentication）

- 使用 Email + 密碼註冊/登入
- Email 需唯一，格式需驗證
- 密碼需至少 8 字元
- 登入後產生 Session/Token
- 每筆帳務與類別需綁定 user_id
- 使用者僅能存取自己的資料
- 登出後清除 Session/Token
- Session 過期需重新登入

### 4.2 帳務紀錄（Transaction）

#### 帳務欄位

| 欄位 | 說明 | 規則 |
|------|------|------|
| type | 類型 | `income` / `expense`，必填 |
| amount | 金額 | 正整數，必填，> 0 |
| category_id | 類別 | FK，必填，需為使用者可用類別 |
| date | 日期 | 必填，格式 YYYY-MM-DD |
| note | 備註 | 選填，最多 200 字 |

#### 帳務規則

- 新增：填寫表單後送出，自動綁定當前使用者
- 編輯：可修改所有欄位，更新後即時反映於列表與報表
- 刪除：需顯示二次確認對話框，確認後永久刪除
- 類別選擇：僅顯示狀態為 active 的類別

### 4.3 類別管理（Category）

#### 預設類別（系統建立）

| 名稱 | 適用類型 |
|------|----------|
| 食物 | expense |
| 生活 | expense |
| 交通 | expense |
| 薪水 | income |
| 提款 | income |

#### 類別欄位

| 欄位 | 說明 | 規則 |
|------|------|------|
| name | 名稱 | 必填，最多 20 字，同使用者內唯一 |
| is_active | 狀態 | boolean，預設 true |
| type | 適用類型 | `income` / `expense` / `both`，預設 `both` |

#### 類別規則

- 使用者可新增自訂類別
- 使用者可編輯類別名稱
- 使用者可停用類別（設 is_active = false）
- 停用類別不可用於新帳務
- 歷史帳務保留原類別（即使類別已停用）
- 類別不可刪除（僅能停用），確保歷史資料完整性
- 預設類別可被停用但不可被刪除

### 4.4 主要頁面需求（Page Requirements）

#### 頁面清單（Page Inventory）

| 頁面名稱 | 用途 | URL/Route |
|----------|------|-----------|
| 登入頁 | 使用者登入 | `/login` |
| 註冊頁 | 新使用者註冊 | `/register` |
| 帳務列表頁 | 查看/管理每日帳務 | `/transactions` |
| 月報表頁 | 查看月統計與圖表 | `/reports` |
| 類別管理頁 | 管理收支類別 | `/categories` |

#### 各頁面責任（Page Responsibilities）

**登入頁 `/login`**
- 用途：驗證使用者身分
- 主要元件：Email 輸入框、密碼輸入框、登入按鈕、註冊連結
- CTA：登入按鈕、前往註冊連結
- 狀態：Loading（登入中）、Error（帳密錯誤/網路錯誤）

**註冊頁 `/register`**
- 用途：建立新使用者帳號
- 主要元件：Email 輸入框、密碼輸入框、確認密碼輸入框、註冊按鈕、登入連結
- CTA：註冊按鈕、前往登入連結
- 狀態：Loading（註冊中）、Error（Email 已存在/格式錯誤/密碼不符）

**帳務列表頁 `/transactions`**
- 用途：顯示所有帳務，依日期分組，並提供 CRUD 操作
- 主要元件：
  - 日期分組區塊（每日一組，顯示日期、當日總收入、當日總支出）
  - 帳務項目列表（類型圖示、類別、金額、備註）
  - 新增帳務按鈕（Floating Action Button 或頁面頂部按鈕）
  - 帳務表單 Modal（新增/編輯共用）
  - 刪除確認 Modal
- CTA：新增帳務、編輯帳務、刪除帳務
- 狀態：
  - Loading：初始載入帳務列表
  - Ready：顯示帳務列表
  - Empty：無任何帳務時顯示空狀態提示
  - Error：API 錯誤時顯示錯誤訊息

**月報表頁 `/reports`**
- 用途：顯示指定月份的收支統計與圖表
- 主要元件：
  - 年月選擇器（可切換年份/月份）
  - 統計卡片（總收入、總支出、淨收支）
  - 類別圓餅圖（支出分布）
  - 每日收支長條圖（X 軸日期，Y 軸金額）
  - 匯出 CSV 按鈕
- CTA：切換月份、匯出 CSV
- 狀態：
  - Loading：載入報表資料
  - Ready：顯示報表與圖表
  - Empty：該月無帳務時顯示空狀態提示
  - Error：API 錯誤時顯示錯誤訊息

**類別管理頁 `/categories`**
- 用途：管理使用者的收支類別
- 主要元件：
  - 類別列表（名稱、狀態、類型）
  - 新增類別按鈕
  - 類別表單 Modal（新增/編輯共用）
  - 停用確認提示
- CTA：新增類別、編輯類別、停用/啟用類別
- 狀態：
  - Loading：載入類別列表
  - Ready：顯示類別列表
  - Empty：無自訂類別時顯示預設類別
  - Error：API 錯誤時顯示錯誤訊息

#### 路由存取控制（Route Access Control）

| 頁面 | Guest | User | 不可進入時行為 |
|------|-------|------|----------------|
| `/login` | ✅ | ✅（但已登入自動導向 `/transactions`） | - |
| `/register` | ✅ | ✅（但已登入自動導向 `/transactions`） | - |
| `/transactions` | ❌ | ✅ | 導向 `/login` |
| `/reports` | ❌ | ✅ | 導向 `/login` |
| `/categories` | ❌ | ✅ | 導向 `/login` |

#### 導覽列規則（Navigation Visibility Rules）

**Guest 狀態（未登入）**
- 顯示：Logo、登入連結、註冊連結
- 隱藏：帳務列表、月報表、類別管理、登出

**User 狀態（已登入）**
- 顯示：Logo、帳務列表、月報表、類別管理、登出
- 隱藏：登入、註冊

#### 全站共用元件責任（Layout Responsibility）

**Header**
- 固定於頁面頂部
- 包含 Logo（點擊導向首頁/帳務列表）
- 依登入狀態顯示對應導覽項目
- 登入/登出入口統一放在 Header 右側
- RWD：手機版收合為漢堡選單

**CTA 去重規則**
- 登入按鈕：僅在 Header 出現，登入頁內以表單送出按鈕為主
- 新增帳務按鈕：僅在帳務列表頁出現，不在 Header
- 匯出 CSV 按鈕：僅在月報表頁出現，不在 Header

### 4.5 帳務列表頁（Daily Grouped List）

- 依「日期」分組顯示，最新日期在最上方
- 每個日期區塊顯示：
  - 日期標題（例：2026/02/01 週日）
  - 當日總收入
  - 當日總支出
- 每筆帳務顯示：
  - 類型圖示（收入/支出）
  - 類別名稱
  - 金額（收入為正，支出為正但顯示時可加負號或紅色）
  - 備註（若有）
- 帳務列表依時間倒序排列（同日內最新在前）

### 4.6 帳務操作規則

- **編輯**：點擊帳務開啟 Modal，修改後即時更新
- **刪除**：點擊刪除按鈕，彈出二次確認 Modal，確認後刪除
- **即時更新**：編輯/刪除後，帳務列表、日統計、月報表、圖表皆需同步更新

### 4.7 月報表（Monthly Report）

#### 時間維度選擇

- 年份選擇器：至少可選當年與前 2 年
- 月份選擇器：1~12 月
- 預設顯示當前年月

#### 統計內容

| 項目 | 計算方式 |
|------|----------|
| 總收入 | SUM(amount) WHERE type = 'income' AND month = selected |
| 總支出 | SUM(amount) WHERE type = 'expense' AND month = selected |
| 淨收支 | 總收入 − 總支出 |

### 4.8 圖表需求（Charts）

#### 4.8.1 類別圓餅圖（支出）

- 僅統計 type = 'expense' 的帳務
- 依 category 分組計算金額
- 顯示各類別佔比（%）
- 點擊可顯示該類別金額
- 空資料時顯示「本月無支出」

#### 4.8.2 月收支長條圖

- X 軸：日期（1~31，僅顯示有資料的日期）
- Y 軸：金額
- 每日顯示兩根柱狀：收入（綠）、支出（紅）
- Hover 顯示詳細數值
- 空資料時顯示「本月無資料」

### 4.9 匯出功能（選擇性）

- 匯出格式：CSV
- 匯出範圍：當前選擇的月份
- CSV 欄位：日期、類型、類別、金額、備註
- 檔名格式：`transactions_YYYY_MM.csv`
- 匯出資料需與畫面顯示一致

## 5. 非功能需求（Non-functional Requirements）

### 5.1 響應式設計（RWD）

- 支援桌機（≥1024px）、平板（768px~1023px）、手機（<768px）
- 手機版 Header 收合為漢堡選單
- 圖表需支援手機橫向捲動或自適應尺寸

### 5.2 狀態處理

| 狀態 | 處理方式 |
|------|----------|
| Loading | 顯示 Skeleton 或 Spinner |
| Error | 顯示錯誤訊息與重試按鈕 |
| Empty | 顯示空狀態插圖與引導文字 |

### 5.3 錯誤處理

| 錯誤碼 | 情境 | 處理 |
|--------|------|------|
| 401 | 未認證/Token 過期 | 導向登入頁 |
| 403 | 無權限存取該資源 | 顯示 403 頁面或提示 |
| 404 | 資源不存在 | 顯示 404 頁面 |
| 5xx | 伺服器錯誤 | 顯示錯誤訊息與重試按鈕 |

### 5.4 資料一致性

- 帳務列表、日統計、月統計、圖表需使用同一資料來源
- CRUD 操作後所有相關顯示需即時同步
- 匯出資料需與當前篩選條件一致

### 5.5 效能

- 帳務列表支援分頁或無限捲動（每頁 30 筆）
- 圖表資料由後端聚合，避免前端大量計算

### 5.6 安全性

- 密碼以 Hash 儲存（bcrypt）
- API 需驗證 Token 與 user_id
- 防止 XSS（輸入轉義）
- 防止 CSRF（Token 驗證）

### 5.7 UI 一致性

- 同一動作的入口在全站一致
- Header 與頁面內不重複相同 CTA
- 按鈕樣式統一（主要/次要/危險）

## 6. 資料模型（Data Model）

### User

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID/Int | PK |
| email | String | Unique, Not Null |
| password_hash | String | Not Null |
| created_at | DateTime | 建立時間 |
| updated_at | DateTime | 更新時間 |

### Category

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID/Int | PK |
| user_id | UUID/Int | FK → User.id, Nullable（系統預設類別為 Null） |
| name | String | Not Null, Max 20 |
| type | Enum | `income` / `expense` / `both` |
| is_active | Boolean | 預設 true |
| is_default | Boolean | 預設 false（系統預設類別為 true） |
| created_at | DateTime | 建立時間 |
| updated_at | DateTime | 更新時間 |

### Transaction

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID/Int | PK |
| user_id | UUID/Int | FK → User.id, Not Null |
| category_id | UUID/Int | FK → Category.id, Not Null |
| type | Enum | `income` / `expense` |
| amount | Int | 正整數, > 0 |
| date | Date | Not Null |
| note | String | Nullable, Max 200 |
| created_at | DateTime | 建立時間 |
| updated_at | DateTime | 更新時間 |

### 關聯

- User 1:N Transaction（一個使用者有多筆帳務）
- User 1:N Category（一個使用者有多個自訂類別）
- Category 1:N Transaction（一個類別對應多筆帳務）
- 系統預設類別 user_id 為 Null，所有使用者共用

### 索引建議

- Transaction: `(user_id, date)` 複合索引（加速日期查詢）
- Transaction: `(user_id, date, type)` 複合索引（加速月報表聚合）
- Category: `(user_id, is_active)` 複合索引（加速類別選擇）
