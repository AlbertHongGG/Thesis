# 人工驗收確認表
線上課程平台（非影音串流）

## 環境與前置條件
- [ ] 測試環境已可正常開啟 `/`、`/courses`、`/login`、`/register`。
- [ ] 準備 4 組帳號：Guest（未登入）、Student、Instructor、Admin。
- [ ] 準備至少 5 門課程資料，狀態至少涵蓋 `draft/submitted/published/rejected/archived`。
- [ ] 準備至少 1 門已上架課程含章節與單元（`text`、`image`、`pdf` 各至少 1 筆）。
- [ ] 準備至少 1 筆 Student 已購買課程資料與 1 筆未購買對照資料。
- [ ] 準備 Instructor A / Instructor B 各自建立的課程，用於跨作者權限驗證。
- [ ] 準備至少 1 筆 `submitted` 課程供 Admin 審核（核准/駁回）。
- [ ] 測試瀏覽器至少涵蓋桌機與手機寬度（例如 390px 與 1280px）。
- [ ] 伺服器時區與顯示時區一致，購買日期與審核時間顯示無明顯偏差。

## 角色與權限邊界
- [ ] Guest 導覽僅顯示課程列表、登入、註冊，不顯示我的課程、教師入口、管理後台入口。
- [ ] Student 導覽顯示課程列表、我的課程、登出；不顯示教師入口與管理後台入口。
- [ ] Instructor 導覽顯示課程列表、我的課程、教師課程管理、登出；不顯示管理後台入口。
- [ ] Admin 導覽顯示課程列表、管理後台入口、登出（我的課程可選顯示）。
- [ ] Guest 直接進入 `/my-courses` 被拒絕（導向登入或 401 對應 UI，且策略一致）。
- [ ] Student 直接進入 `/admin/review`、`/admin/courses`、`/admin/taxonomy`、`/admin/users`、`/admin/stats` 皆顯示 403。
- [ ] Instructor 直接進入 `/admin/*` 皆顯示 403。
- [ ] 非 Instructor / Admin 進入 `/instructor/*` 皆顯示 403。
- [ ] 其他人嘗試查看非公開課程（`draft/submitted/rejected/archived`）詳情時顯示 404，不暴露資源存在性。
- [ ] 未購買且非作者且非管理員進入課程閱讀頁 `/my-courses/:courseId` 顯示 403。

## 端到端主流程

### Student 主流程
- [ ] 可從 `/courses` 瀏覽已上架課程清單（含封面、標題、價格、分類）。
- [ ] 可進入 `/courses/:courseId` 查看課程行銷資訊與章節/單元大綱標題。
- [ ] 未購買時可看到購買入口，且看不到單元內容與 PDF 下載。
- [ ] 購買成功後可進入 `/my-courses`，並看到該課程與購買日期。
- [ ] 可進入 `/my-courses/:courseId` 閱讀單元內容（文字/圖片/PDF）。
- [ ] 可標記單元完成，完成數與進度百分比即時更新。

### Instructor 主流程
- [ ] 可在 `/instructor/courses/new` 建立課程，初始狀態為 `draft`。
- [ ] 可在 `/instructor/courses/:courseId/edit` 編輯課程基本資訊（標題、描述、分類、標籤、價格、封面）。
- [ ] 可在 `/instructor/courses/:courseId/curriculum` 新增章節與單元並調整順序。
- [ ] 可提交審核使狀態 `draft/rejected -> submitted`。
- [ ] 課程上架後可維護內容，並可在 `published <-> archived` 間切換。

### Admin 主流程
- [ ] 可在 `/admin/review` 查看 `submitted` 清單。
- [ ] 可核准課程使狀態 `submitted -> published`。
- [ ] 可駁回課程使狀態 `submitted -> rejected`，且駁回理由必填。
- [ ] 可在 `/admin/courses` 管理上架/下架（`published <-> archived`）。
- [ ] 可在 `/admin/taxonomy` 管理分類與標籤（新增/編輯/停用）。
- [ ] 可在 `/admin/users` 管理帳號啟用狀態與主要角色。
- [ ] 可在 `/admin/stats` 查看平台統計（課程狀態數、購買數、使用者數）。

## 全站狀態品質（Loading / Error / Empty / Retry）
- [ ] 主要頁面（`/courses`、`/my-courses`、`/instructor/*`、`/admin/*`）進入時有 loading 狀態。
- [ ] 送出型操作（登入、註冊、建立課程、儲存編輯、提交審核、核准、駁回、購買、標記完成）請求中按鈕為 disabled。
- [ ] 列表 API 失敗時顯示 Error 狀態，不可誤顯示 Empty。
- [ ] 空資料時顯示 Empty 狀態與可理解提示（不是白畫面）。
- [ ] Error 狀態提供 Retry，重試可再次觸發請求並回復正常狀態。
- [ ] 發生錯誤後頁面不會永遠停在 loading。

## 錯誤碼與導向
- [ ] 401 情境（session 失效或未登入受保護頁）導向登入或顯示 401 對應 UI，行為一致。
- [ ] 403 情境（權限不足）顯示 403 頁或明確權限不足訊息。
- [ ] 404 情境（未知路由、不可見課程）顯示 404 並提供回到可用頁面的入口。
- [ ] 5xx 情境顯示錯誤頁或全站錯誤提示，並提供可重試路徑。
- [ ] 受保護 API 在前端被擋下時，後端仍回傳對應 401/403，不僅靠前端限制。

## RWD 與可用性
- [ ] 手機寬度下 Header 不會出現重複登入/登出入口。
- [ ] 手機寬度下課程卡與主要 CTA 可點擊且不被遮擋。
- [ ] 手機寬度下 `/my-courses` 進度資訊可讀且不截斷關鍵數值。
- [ ] 手機寬度下 Instructor 課程管理頁操作入口（建立、編輯、課綱、提交）可完成。
- [ ] 手機寬度下 Admin 審核頁核准/駁回操作可完成且不誤觸。
- [ ] 桌機與手機顯示的權限規則一致（不因裝置而多出不該出現入口）。

## 功能需求驗收

### 4.1 帳號與認證（Email + Password）
- [ ] 可用唯一 Email + 合法密碼完成註冊。
- [ ] 重複 Email 註冊失敗並顯示可理解錯誤。
- [ ] 密碼長度不足時註冊失敗並顯示欄位錯誤。
- [ ] 可用正確帳密登入並建立 session。
- [ ] 錯誤帳密登入失敗並顯示錯誤訊息。
- [ ] 登出可清除 session，導覽切回 Guest 狀態。
- [ ] session 失效後進入受保護頁面會觸發 401 對應行為。
- [ ] 認證狀態在 UI 與 API 一致（UI 顯示已登入時，受保護 API 可通過；登出後不可通過）。

### 4.2 角色權限（RBAC）與路由存取控制
- [ ] Student 只能進入其允許頁面，不可進入 `/instructor/*` 與 `/admin/*`。
- [ ] Instructor 可進入 `/instructor/*`，不可進入 `/admin/*`。
- [ ] Admin 可進入所有 `/admin/*` 與教師管理相關頁面。
- [ ] 前端會隱藏不該出現的入口，不以「顯示後再擋」替代。
- [ ] 直接呼叫未授權 API 會被後端拒絕（403/404 策略符合規格）。
- [ ] 課程閱讀權限判斷在 UI、路由與 API 三層一致。

### 4.3 課程（Course）

#### 4.3.1 課程欄位
- [ ] 課程可正確保存與顯示標題、描述、分類、標籤、價格、封面、狀態。
- [ ] 價格小於 0 無法保存並顯示錯誤。
- [ ] 停用中的分類/標籤不可被新設定到課程。
- [ ] 課程欄位更新後，列表與詳情顯示一致。

#### 4.3.2 課程狀態機
- [ ] 可執行合法轉換：`draft -> submitted`。
- [ ] 可執行合法轉換：`submitted -> published`。
- [ ] 可執行合法轉換：`submitted -> rejected`（需理由）。
- [ ] 可執行合法轉換：`rejected -> draft`。
- [ ] 可執行合法轉換：`published -> archived`。
- [ ] 可執行合法轉換：`archived -> published`。
- [ ] 非法轉換會被拒絕並回傳可理解錯誤（HTTP 400）。
- [ ] 非作者不得變更課程狀態（管理員除外）。
- [ ] 狀態變更後，Instructor 列表、Admin 列表與課程詳情顯示一致。

#### 4.3.3 課程可見性規則（Marketing vs Content）
- [ ] 所有人可瀏覽 `published` 課程行銷資訊。
- [ ] 非作者/非管理員不可瀏覽 `draft/submitted/rejected/archived` 課程詳情（404）。
- [ ] 未購買者在課程詳情僅可看到章節/單元標題，不可看到內容。
- [ ] 作者、購買者、管理員可進入課程內容。
- [ ] PDF 下載連結受保護，未授權者不可下載。

### 4.4 章節與單元（Curriculum）
- [ ] 可新增章節並設定順序。
- [ ] 可新增單元並設定順序。
- [ ] 單元內容型態可設定為 `text`、`image` 或 `pdf`。
- [ ] 可編輯章節/單元資料並保存成功。
- [ ] 可刪除章節/單元並反映在大綱。
- [ ] 排序調整後重新載入仍維持一致順序。
- [ ] 非作者且非管理員不可新增/編輯/刪除章節與單元。
- [ ] 內容更新後，課程閱讀頁顯示與課綱編輯結果一致。

### 4.5 課程購買（Purchase）
- [ ] Student 可購買 `published` 課程並建立購買紀錄。
- [ ] Instructor 以學習者身份可購買他人 `published` 課程。
- [ ] 已購買同一課程時再次購買會被阻擋並提示「已購買」。
- [ ] 非 `published` 課程不可購買。
- [ ] 購買成功後 `/my-courses` 立即可見該課程。
- [ ] 購買後可永久存取，不因重新登入而遺失權限。
- [ ] 購買狀態在課程詳情 CTA、我的課程列表、閱讀權限三處一致。

### 4.6 存取控制（Access Control）
- [ ] 課程作者可讀取自己課程內容（未購買亦可）。
- [ ] 購買者可讀取已購買課程內容。
- [ ] Admin 可強制存取任何課程內容。
- [ ] 未購買且非作者且非管理員進入閱讀內容被拒絕（403）。
- [ ] 權限變更後（例如角色調整）讀取結果立即反映。

### 4.7 我的課程（My Courses）
- [ ] 僅顯示目前使用者已購買課程，不顯示他人資料。
- [ ] 每門課顯示封面、標題、講師、購買日期。
- [ ] 顯示進度為「完成單元數 / 總單元數」。
- [ ] 單元完成標記後，我的課程進度同步更新。
- [ ] 無已購買課程時顯示 Empty 與前往 `/courses` 入口。

### 4.8 管理後台（Admin Panel）

#### 課程審核
- [ ] `/admin/review` 只顯示 `submitted` 課程。
- [ ] 核准成功後課程狀態改為 `published` 並自待審清單移除。
- [ ] 駁回必填理由，成功後課程狀態改為 `rejected`。
- [ ] 審核失敗時課程狀態不應部分更新。
- [ ] 每次審核皆寫入審核紀錄（誰、何時、決策、理由）。

#### 課程上下架管理
- [ ] `/admin/courses` 可切換 `published <-> archived`。
- [ ] 非 `published/archived` 狀態不提供上下架操作。
- [ ] 切換後狀態在 Admin 與 Instructor 視圖一致。

#### 分類與標籤管理
- [ ] 可新增分類與標籤。
- [ ] 可編輯分類與標籤名稱。
- [ ] 可停用分類與標籤。
- [ ] 名稱唯一性衝突時保存失敗並提示。
- [ ] 停用後不再可供新課程選用。

#### 使用者管理
- [ ] `/admin/users` 顯示 email、角色、啟用狀態。
- [ ] 可更新使用者主要角色（student/instructor/admin）。
- [ ] 可停用與啟用帳號。
- [ ] 被停用帳號登入會被拒絕。
- [ ] 角色更新後導覽與可訪問頁面立即符合新角色。

#### 平台統計
- [ ] `/admin/stats` 顯示課程數量（依狀態）、購買數量、使用者數量。
- [ ] 統計數值可追溯到現有資料，不出現明顯不一致。
- [ ] 統計刷新後數值更新邏輯一致，不出現舊值殘留。

### 4.9 主要頁面與導覽規則
- [ ] Page Inventory 中每個頁面皆可由合法角色進入。
- [ ] 各頁主要 CTA 位於頁面內容區，不與 Header 重複。
- [ ] Header 與頁面內不出現重複登入/登出 CTA。
- [ ] `/courses/:courseId` 與 `/my-courses/:courseId` 對同一課程的可見資訊符合權限規格。
- [ ] `/403`、`/404`、`/500` 頁可被正確導向並提供回復路徑。

## 資料一致性檢查
- [ ] 建立課程後，Instructor 課程列表可見該課程，詳情頁資料一致。
- [ ] 編輯課程基本資訊後，課程列表卡片與課程詳情內容一致。
- [ ] 課綱更新後，閱讀頁章節/單元順序與內容一致。
- [ ] 狀態轉換後，Instructor 列表、Admin 列表、課程詳情三處狀態一致。
- [ ] 購買成功後，課程詳情 CTA、我的課程、閱讀權限一致。
- [ ] 標記單元完成後，閱讀頁與我的課程進度一致。
- [ ] 分類/標籤停用後，課程編輯頁不可再選用，既有課程顯示策略一致。
- [ ] 使用者角色或啟用狀態變更後，導覽、路由、API 權限判定一致。
- [ ] Admin 統計與基礎資料（課程、購買、使用者）在同一時間點一致。

## 非功能需求補充驗收

### 安全
- [ ] 未授權者不可透過 URL 直接取得課程內容。
- [ ] 課程文字內容顯示採安全渲染，避免注入風險。
- [ ] 受保護檔案（PDF）下載需通過授權檢查。

### 可靠性
- [ ] 重要操作在網路失敗後可重試，不造成重複副作用。
- [ ] 重整頁面後，登入狀態與導覽可正確恢復。
- [ ] 重整後進度與課程狀態不出現回滾或不一致。

### 可用性
- [ ] 主要錯誤訊息可被一般使用者理解並可採取下一步。
- [ ] 空狀態文案可引導使用者返回可操作頁面。
- [ ] 重要流程（購買、提交審核、核准/駁回）都有明確成功/失敗回饋。

## 分角色覆蓋（可做 / 不可做）

### Guest
- [ ] 可做：瀏覽 `/courses`、查看已上架課程詳情、註冊、登入。
- [ ] 不可做：進入 `/my-courses`、閱讀課程內容、進入 `/instructor/*`、進入 `/admin/*`。

### User（已登入共通）
- [ ] 可做：可存取需要登入的使用者區域（至少包含 `/my-courses`）並維持有效 session。
- [ ] 可做：可在自己的授權範圍內讀取與操作資料。
- [ ] 不可做：不可存取未授權路由或未授權資源（應回 403 或 404，依規格策略）。

### Student
- [ ] 可做：購買已上架課程、查看我的課程、閱讀已購買課程、標記單元完成。
- [ ] 不可做：建立/編輯課程、提交審核、管理分類標籤、管理使用者、審核課程。

### Instructor
- [ ] 可做：建立與管理自己的課程、維護課綱、提交審核、上下架自己的課程。
- [ ] 不可做：審核課程、管理全站分類標籤、管理全站使用者、進入 `/admin/*`。

### Admin
- [ ] 可做：審核課程、管理課程上下架、管理分類標籤、管理使用者、查看統計、強制存取課程內容。
- [ ] 不可做：無（仍須符合操作合法性與資料驗證規則）。

## 跨帳號資料隔離
- [ ] Student A 不能看到 Student B 的 `my-courses` 清單。
- [ ] Student A 不能閱讀 Student B 未購買但可閱讀的課程內容（若自己未購買則 403）。
- [ ] Instructor A 不能編輯 Instructor B 的課程（應回 404）。
- [ ] 非 Admin 不可修改其他使用者角色或啟用狀態。
